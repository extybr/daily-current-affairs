Инструкция по tail

Основные опции:
tail файл — последние 10 строк файла
tail -n число — последние N строк
tail -c число — последние N байт
tail -f — следить за изменениями файла (follow)
tail -F — следить с переоткрытием файла

Просмотр строк:
tail -20 — последние 20 строк
tail -n +20 — начиная с 20-й строки до конца
tail -n -20 — последние 20 строк (аналогично -20)

Слежение за файлами:
tail -f лог.файл — следить за логом в реальном времени
tail -F лог.файл — следить, даже если файл пересоздаётся
tail -f --pid=PID — прекратить слежение при завершении процесса

Несколько файлов:
tail файл1 файл2 — последние строки нескольких файлов
tail -q — не выводить заголовки файлов (quiet)
tail -v — всегда выводить заголовки файлов (verbose)

Примеры:
tail /var/log/syslog — последние 10 строк системного лога
tail -n 100 access.log — последние 100 строк лога доступа
tail -c 512 data.bin — последние 512 байт бинарного файла
tail -f /var/log/auth.log — следить за логом аутентификации
tail -F /tmp/app.log — следить за логом, который может ротироваться
tail -n +50 file.txt — строки с 50-й до конца
tail -n 0 -f log.txt — следить только за новыми строками
tail -n +2 data.csv — пропустить заголовок CSV
tail -q *.log — просмотреть все логи без заголовков
tail -v server*.log — просмотреть с явными заголовками
tail --retry file.log — продолжать попытки если файл недоступен

Комбинации:
tail -f log.txt | grep ERROR — фильтровать новые строки с ERROR
tail -f log.txt > output.txt — записывать новые строки в файл
ps aux | tail -n +2 — пропустить заголовок ps
dmesg | tail -20 — последние 20 строк ядра
history | tail -10 — последние 10 команд из истории

Слежение за несколькими файлами:
tail -f log1.log log2.log — следить за двумя логами одновременно
tail -F /var/log/*.log — следить за всеми логами в директории

Специальные опции:
tail --max-unchanged-stats=N — для ротируемых логов
tail --sleep-interval=N — интервал проверки для -f (секунды)

Потоковый ввод:
cat file.txt | tail -n 5 — последние 5 строк из потока
generator.sh | tail -f — следить за выводом команды

Практическое использование:
tail -f /var/log/nginx/access.log — мониторинг веб-доступа
tail -f /var/log/syslog | grep sshd — следить за SSH подключениями
tail -n 1000 large.log | less — просмотр конца большого файла
tail -c +100 file.bin — пропустить первые 100 байт

Вместе с head:
head -20 file.txt | tail -10 — строки 11-20
sed -n '10,20p' file.txt — аналогичный результат

Ротация логов:
tail -F — предпочтительнее для логов, которые ротируются (rotate)
tail -f — может потерять новые данные при ротации

Отладка:
tail --version — версия tail
tail --help — справка
