
конвертирование в gif
	ffmpeg -i input.mkv -ss 00:02:27 -to 00:02:29 output.gif

обрезка и конвертирование
	ffmpeg -y -ss 143 -to 150 -i input.webm output.mkv
конвертирование и обрезка от 143 сек до 171 сек
	ffmpeg -y -ss 143 -to 171 -i input.mkv output.mkv

обрезка сначала до 467 сек, !без конвертации
	ffmpeg -to 467 -i input.mp4 -c copy output.mp4

конвертирование
	ffmpeg -i input.webm -c copy output.mkv

FFmpeg масштабирование 50% по горизонтали и вертикали, кодируем стандартным кодеком
	ffmpeg -i input.mpg -vf scale=iw*0.5:ih*0.5 output.mp4

масштабирование указано в пикселях (половина от оригинального 720×576)
ffmpeg -i input.mpg -vf scale=360:288 output.mp4

Разрешение
Изменить разрешение (размер) видео: -s
Формат: widthxheight
	ffmpeg -i input_4k.mp4 -s 1920x1080 output_1080p.mp4
	ffmpeg -i input_4k.mp4 -s 1280x720 output_720p.mp4
Изменять можно не только разрешение видео, но также и картинок:
	ffmpeg -i input.png -s 320x240 result.png

Частота кадров
Изменить частоту кадров в секунду: -r
	ffmpeg -i input.mp4 -r 25 result.mp4

Убрать аудио: -an
Убрать видео: -vn
	ffmpeg -i input.mkv -an result.mp4
	ffmpeg -i input.mkv -vn result.mp3

БитрейтИзменить битрейт аудио: -b:a
Изменить битрейт видео: -b:v
Битрейт указывается числом в битах в секунду, но для удобства можно использовать формат:
123k — 123 кбпс, 2M — 2 Мбпс
	ffmpeg -i input.mkv -vn -b:a 320k result.mp3
	ffmpeg -i input.flac -b:a 128k result.mp3
	ffmpeg -i input.mkv -b:v 16M result.mp4
	ffmpeg -i input.mkv -b:v 16M -b:a 192k result.mp4
Ещё примеры:
масштабирование видео указано в битрейтах
	ffmpeg -i input.mp4 -b:v 3000k output.mp4
вырезаем audio с указанным битрейтом
	ffmpeg -i input.mp4 -b:a 144k output1.mp3

с контролем качества
Размер видео уменьшился более чем в 5 раз без ощутимой потери качества.
Опция -qscale:v n, сокращенно -q:v n позволяет установить уровень качества 
генерируемого видеопотока, где n принимает значения в интервале от 1 до 31. 
Значение 1 соответствует самому лучшему качеству, а 31 — самому худшему.
	ffmpeg -i MVI_4703.MOV -q:v 4 MVI_4703.avi

склеивание видео
	ffmpeg -f concat -i concat.txt -c copy output.mp4
содержимое concat.txt:
file File1.mkv
file File2.mkv

вырезка фрагмента и склейка двух частей
	input="my_video_file.mp4" && output_1="part_1.mp4" && output_2="part_2.mp4" &&
	ffmpeg -i $input -to 00:00:05 -c copy $output_1 &&
	ffmpeg -i $input -ss 00:00:20 -c copy $output_2 &&
	touch concat.txt && echo "file '$output_1'\nfile '$output_2'" > concat.txt &&
	ffmpeg -f concat -i concat.txt -c copy output.mp4

Копирование потоков. Сохранить оригинальные кодеки и скопировать их как есть:
-c copy — сохранить видеокодек и аудиокодек,
-c:v copy — сохранить только видеокодеки,
-c:a copy — сохранить только аудиокодеки.
В этом случае не будет происходить перекодирование, что позволяет быстрее обработать файл.
Это может быть полезно для случаев, когда нужно быстро скачать видео:
    ffmpeg -i "https://sample-videos.com/video123/mp4/480/big_buck_bunny_480p_10mb.mp4" -c copy out.mp4
Или получить аудиодорожку из видеофайла:
    ffmpeg -i in.mp4 -c:a copy out.aac
* Форматы должны быть совместимы. В mp4 для аудио обычно используется кодек aac, 
поэтому и копировать его можно только в aac. Если указать в выходном файле .mp3, то будет ошибка:
    ffmpeg -i in.mp4 -c:a copy out.mp3  => FAIL
Ещё примеры:
    ffmpeg -i input.mkv -c:v copy result.mp4
    ffmpeg -i input.mkv -c:a copy result.mp4

Формат пикселей
Формат пикселейНекоторые смартфоны и браузеры не поддерживают форматы пикселей YUV444, YUV422, 
а работают только с YUV420. Чтобы изменить формат пикселей используется ключ -pix_fmt формат
	ffmpeg -i input.mp4 -pix_fmt yuv420p result.mp4
	
Параметр -r, который задаёт значение FPS (кадров в секунду) видео.
Задать кодек: -c:v libx265
    ffmpeg -i input.mp4 -c:v libx265 -r 60 output.mp4
    
Основные способы поворота (с перекодированием)
Используйте фильтр transpose в параметре -vf:
    90° по часовой стрелке:
    ffmpeg -i input.mp4 -vf "transpose=1" -c:a copy output.mp4
    90° против часовой стрелки:
    ffmpeg -i input.mp4 -vf "transpose=2" -c:a copy output.mp4
    180° (перевернуть вверх ногами):
    ffmpeg -i input.mp4 -vf "transpose=2,transpose=2" -c:a copy output.mp4
    90° по часовой + зеркальное отражение:
    ffmpeg -i input.mp4 -vf "transpose=3" -c:a copy output.mp4

ffmpeg -i input.mkv -c:v libx264 -crf 28 -preset slow output.mp4
    -i input.mkv - входной файл (mkv, avi, mov, mp4 и др.)
    -c:v libx264 - кодек H.264 - работает на любом CPU
    -crf 28 - качество:0 (лучшее) -> 51 (сильное сжатие) 

ffmpeg -i input.mkv -c:v hevc_nvenc -preset p6 -cq 26 output.mp4
    -c:v hevc_nvenc H.265 - баланс скорости и сжатия
    -cq - аналог CRF, но для GPU-кодеков

ffmpeg -i input.mkv -c:v av1_nvenc -preset p7 -tune uhq -rc vbr -cq 35 -multipass fullres output.mp4
    -c:v av1_nvenc AV1 - лучшее сжатие, нужна RTX 40XX
    -multipass fullres - два прохода, лучше качество

___________________  склеить видеофайлы без ошибок  ___________________

Чтобы склеить видеофайлы через ffmpeg без ошибок, перекодирования (потерь) и артефактов (режим concat),
    исходные файлы должны быть почти идентичными по ряду параметров. Вот полный список факторов,
    которые реально влияют на успешность склейки:

Критические факторы (различие гарантированно вызовет ошибку или артефакты):

    1. Видеокодек – должен быть строго одинаковым (например, только H.264, только HEVC).
    2. Аудиокодек – строго одинаковый (AAC, MP3, PCM и т.д.).
    3. Размер кадра (разрешение видео) – ширина и высота в пикселях должны совпадать полностью.
    4. Частота кадров (fps) – должна быть идентичной. Разные fps при склейке без перекодирования приведут к рассинхронизации.
    5. Формат пикселей (pixel format) – например, yuv420p, yuvj420p, yuv422p – должны совпадать.
    6. Количество аудиоканалов (моно, стерео, 5.1) – должно быть одинаковым.
    7. Частота дискретизации аудио (sample rate) – 44100, 48000 Гц и т.д.
    8. Битрейт аудио – должен совпадать (хотя иногда вариации в пределах одного кодека проходят, но лучше одинаковый).
    9. Формат контейнера (MP4, MKV, TS) – для безопасной склейки через concat demuxer лучше использовать TS (MPEG-TS) или
       предварительно перепаковывать в промежуточный формат.

Важные, но иногда прощаемые (могут привести к микро-артефактам):

    1. Битрейт видео – если видеокодеки и все остальное совпадают, битрейт может отличаться.
       При склейке без перекодирования потоки просто соединяются, битрейт не должен мешать,
       но может повлиять на конечный VBR-профиль. Однако наличие разных GOP-структур может дать артефакты на стыках.
    2. GOP (Group of Pictures) – длина GOP и расположение IDR-кадров.
       Если файл нарезан не по IDR-кадрам, на стыке будет "каша".
       Поэтому рекомендуют предварительно переконвертировать каждый файл в -g 1 (все кадры – keyframes) или 
       использовать формат mkv с ключом +genpts.
    3. Равномерность времени (PTS/DTS) – должны быть непрерывными.
       При склейке ffmpeg исправляет PTS, но если в файлах есть нестандартные задержки, возможны сбои.

Что не играет роли (при склейке без перекодирования):

    1. Имена файлов
    2. Метаданные (теги, дата создания)
    3. Битрейт контейнера (в mkv/mp4 он не фиксирован)
    4. Наличие B-кадров (если кодеки одинаковые)

Итоговая таблица приоритета:
   Фактор	                       Критичность
Видеокодек                      ✅ Обязательно одинаковый
Аудиокодек                      ✅ Обязательно одинаковый
Разрешение                      ✅ Одинаковое
FPS                             ✅ Одинаковая
Pixel format                    ✅ Одинаковый
Кол-во аудиоканалов             ✅ Одинаковое
Частота дискретизации аудио     ✅ Одинаковая
Битрейт аудио                   ⚠️ Желательно одинаковый
Битрейт видео                   ❌ Не критичен
Длина GOP                       ⚠️ Важен для качества стыков

Если какой-то параметр отличается – без перекодирования не обойтись.
    Придется сделать ffmpeg -i input1 -i input2 -filter_complex concat=n=2:v=1:a=1 -c:v libx264 ... с потерей качества.

Рекомендуемый способ склейки без потерь:

    #!/bin/bash

    # 1. Перепаковать всё в MPEG-TS (выравнивает PTS/DTS)
    for f in *.mp4; do
      ffmpeg -i "$f" -c copy -bsf:v h264_mp4toannexb -f mpegts "temp_$f.ts"
    done

    # 2. Склеить через concat демаксер
    for f in temp_*.ts; do
      echo "file '$f'" >> list.txt
    done
    ffmpeg -f concat -safe 0 -i list.txt -c copy output.mp4
    rm -f list.txt
