Инструкция по ln

Создание ссылок:
ln цель имя — создать жесткую ссылку
ln -s цель имя — создать символическую ссылку

Жесткие ссылки (hard links):
ln file1 file2 — file2 ссылается на те же данные что file1
ln -f — принудительно перезаписать существующую ссылку
ln -i — запросить подтверждение при перезаписи

Символические ссылки (soft/symlinks):
ln -s /путь/к/цели ссылка — создать симлинк
ln -sf — перезаписать существующий симлинк
ln -si — подтверждение для симлинков

Директории:
ln -s директория/ ссылка — симлинк на директорию
ln -d — разрешить жесткие ссылки на директории (только для root)
ln -F — принудительное создание ссылок в директориях

Относительные пути:
ln -sr цель ссылка — создать симлинк с относительным путем

Массовое создание:
ln -s цель1 цель2 ... директория/ — создать несколько ссылок в директории

Информация:
ln -v — подробный вывод
ln -t директория — создать ссылки в указанной директории

Примеры:
ln original.txt link.txt — жесткая ссылка
ln -s /usr/bin/python3 python — симлинк на python3
ln -sf /new/version /usr/bin/app — обновить симлинк
ln -si old new — с подтверждением перезаписи
ln -s ../configs/app.conf . — относительный симлинк
ln -sv /etc/nginx/sites-available/* /etc/nginx/sites-enabled/ — массовое создание симлинков
ln -t ~/bin /usr/bin/program — создать ссылку в ~/bin

Различия жестких и символических ссылок:
Жесткие:
    Тот же inode
    Нельзя на директории (обычно)
    Работают только в одной ФС
    
Символические:
    Разные inode
    Можно на директории
    Можно между разными ФС
    Содержат путь к цели

Проверка ссылок:
ls -l — показывает куда ведут симлинки
readlink ссылка — показать цель симлинка
stat ссылка — информация о ссылке

Удаление:
rm ссылка — удаляет ссылку (не цель)
unlink ссылка — удалить ссылку
