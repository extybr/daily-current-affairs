Инструкция по xxd

Основные опции:
xxd файл — hex дамп файла
xxd -b файл — бинарный дамп (биты)
xxd -g байты — группа байтов (по умолчанию 2)
xxd -l длина — ограничить длину вывода
xxd -s смещение — начать со смещения

Обратное преобразование:
xxd -r — обратное преобразование hex в бинарный
xxd -p — простой hex вывод (без адресов и ASCII)
xxd -r -p — обратное преобразование простого hex

Вывод:
xxd -u — заглавные hex цифры
xxd -c колонки — байтов на строку (по умолчанию 16)
xxd -e — little-endian порядок байтов
xxd -i — вывод как C массив

Примеры:
xxd file.bin — hex дамп файла
xxd -l 256 file.bin — первые 256 байтов
xxd -s 1024 file.bin — начать с байта 1024
xxd -s -256 file.bin — последние 256 байтов
xxd -g 1 file.bin — по одному байту в группе
xxd -g 4 file.bin — по 4 байта в группе
xxd -u file.bin — заглавные hex символы
xxd -c 32 file.bin — 32 байта на строку
xxd -b file.bin — бинарный вывод (биты)
xxd -p file.bin — простой hex без форматирования
xxd -i file.bin — вывод как C массив
xxd -e file.bin — little-endian (для x86)

Обратное преобразование:
xxd file.bin > hex.txt — создать hex дамп
xxd -r hex.txt > restored.bin — восстановить из hex
echo "48656c6c6f" | xxd -r -p — Hello из hex строки
xxd -p file.bin | tr -d '\n' — одна строка hex

Смещения:
xxd -s 0x100 file.bin — начать с адреса 0x100
xxd -s +0x100 file.bin — пропустить 256 байтов

Работа с stdin/stdout:
cat file.bin | xxd — дамп из stdin
echo "test" | xxd — hex дамп строки
xxd -r hex.txt | xxd — round trip преобразование

Специальные форматы:
xxd -l 16 -ps -c 16 /dev/urandom — 16 случайных байтов одной строкой
xxd -g 2 -u -c 16 file.bin — по 2 байта, заглавные, 16 на строку

Поиск:
xxd file.bin | grep "паттерн" — поиск в hex дампе

Патчинг:
    echo "0000010: 4865 6c6c 6f" | xxd -r - file.bin
    
— заменить байты по адресу 0x10

Пример C массива:
xxd -i data.bin > data.h — создать заголовочный файл с данными

Количество строк:
xxd -l 512 -c 32 file.bin — 512 байтов по 32 на строку = 16 строк

Сравнение файлов:
diff <(xxd file1) <(xxd file2) — сравнить hex дампы

Просмотр бинарных файлов:
xxd executable | head -20 — первые 20 строк hex дампа

Адресация:
Адреса показываются в шестнадцатеричном виде слева
Правая часть — ASCII представление (точка для непечатных)

Маленькие файлы:
xxd -c 8 small.bin — меньше байтов на строку для маленьких файлов
