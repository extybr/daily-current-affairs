Конфигурация конвейера GitLab CI/CD включает в себя:

Глобальные ключевые слова , настраивающие поведение конвейера:

Ключевое слово	Описание
default	Пользовательские значения по умолчанию для ключевых слов задания.
include	Импортируйте конфигурацию из других файлов YAML.
stages	Названия и порядок этапов конвейера.
variables	Определите переменные CI/CD для всех заданий в конвейере.
workflow	Контролируйте, какие типы трубопроводов работают.
Ключевые слова заголовка

Ключевое слово	Описание
spec	Определите спецификации для внешних файлов конфигурации.
Задания , настроенные с использованием ключевых слов задания :

Ключевое слово	Описание
after_script	Переопределить набор команд, которые выполняются после задания.
allow_failure	Разрешить сбой задания. Неудачное задание не приводит к сбою конвейера.
artifacts	Список файлов и каталогов, которые можно прикрепить к заданию в случае успеха.
before_script	Переопределить набор команд, которые выполняются перед заданием.
cache	Список файлов, которые следует кэшировать между последующими запусками.
coverage	Настройки покрытия кода для данного задания.
dast_configuration	Используйте конфигурацию из профилей DAST на уровне задания.
dependencies	Ограничьте количество артефактов, передаваемых конкретному заданию, предоставив список заданий, из которых можно получить артефакты.
environment	Имя среды, в которой развертывается задание.
extends	Записи конфигурации, от которых наследует это задание.
image	Используйте образы Docker.
inherit	Выберите, какие глобальные настройки по умолчанию наследуют все задания.
interruptible	Определяет, можно ли отменить задание, если оно становится избыточным в результате нового запуска.
needs	Выполняйте задания раньше этапа заказа.
pages	Загрузите результат задания для использования с GitLab Pages.
parallel	Сколько экземпляров задания должно выполняться параллельно.
release	Поручает бегуну создать объект выпуска .
resource_group	Ограничьте параллелизм работ.
retry	Когда и сколько раз задание может быть автоматически повторено в случае сбоя.
rules	Список условий для оценки и определения выбранных атрибутов задания, а также того, создано оно или нет.
script	Сценарий оболочки, выполняемый бегуном.
secrets	CI/CD секреты, необходимые для работы.
services	Используйте образы сервисов Docker.
stage	Определяет этап работы.
tags	Список тегов, которые используются для выбора бегуна.
timeout	Определите настраиваемый тайм-аут на уровне задания, который имеет приоритет над настройкой для всего проекта.
trigger	Определяет триггер нисходящего конвейера.
variables	Определите переменные задания на уровне задания.
when	Когда запускать задание.
Глобальные ключевые слова
Некоторые ключевые слова не определены в задании. Эти ключевые слова управляют поведением конвейера или импортируют дополнительную конфигурацию конвейера.
